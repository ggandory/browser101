자주쓰는 태그

<h1>, <h2>, ....  : 글자 크기

<a href="링크 주소"> : 앵커, 링크 연결

<p> : 단락 생성

<ul> / <ol> + <li> :  ul은 순서x, ol은 순서대로 목록 생성

<img src="이미지 주소"> : 이미지 생성

(이미지가 같은 폴더내에 있을 경우엔 이미지 파일명만 입력, 다른 폴더에 있을 경우엔 폴더명/파일순으로 입력)

<img src="", alt="이미지 이름"> : alt안에 텍스트는 브라우저에 나오진 않지만 검색할때, 엑박 뜰 때 보임

<!--블라블라--> : 주석(ctrl + / 단축키)

\*Inline : 집어 넣은 내용과 같은 줄에 위치

\*block : 집어 넣은 내용이 한 줄을 모두 차지(다음 내용은 다음 줄에 위치)

<div> : 블록 생성(div 안에 있는 내용은 하나의 블록으로 묶임)

<span> : 인라인요소, 공통된 것들을 묶어줌(class를 이용해 CSS사용이 편해짐)

<hr> : 수평선 생성태그(닫는 태그 없음)

<br> : 줄바꿈(닫는 태그 없음)

<sup> : 위첨자로 바꿔줌

<sub> : 아래첨자로 바꿔줌

\*Entity code(엔티티코드) : &로 시작 / HTML에서 쓸 수 있는 특별한 코드

예를 들어, HTML에서 '<'처럼 쓸 수 없는 글자가 있다(예약문자, <는 태그의 여는 요소로 이미 예약되어 있음)

이런 쓸 수 없는 문자들을 엔티티코드로 입력하면 화면엔 <로 출력되어짐

\*Semantic code(시멘틱 코드) : 의미론적인 코드(특별한 기능은 수행하지 않고 이름으로만 의미를 가지는 코드)

<div>, <span>을 대신해서 사용할 수 있다.

<main> : 문서의 주요 내용을 의미(반복되는 요소는 제외한다)

<nav> : 페이지에서 네비게이션 링크를 제공하는 것들(페이지 내에서 다른 위치로 이동/ 전혀다른 출처로 이동)

원하는 내용 + 링크를 입력

<section> : 웹 사이트나 앱의 독립적인 부분을 의미

<article> : 문서의 독립적인 부분(독립적인 구성), Section > article

<aside> : 반드시 있어야 하진 않음. 사이드바, 말풍선등으로 표현할 때 사용

<header>, <footer> : 문서의 머릿말, 꼬릿말

<time> : 인라인요소, 시간에 해당하는 곳 옆에 사용

<datetime> : 인라인요소, 오늘 날짜에 해당하는 곳에 사용

<figure> : 구체적인 의미(일러스트, 다이어그램 등)

이 외에도 많으며 주로 개발자들이 보기만해도 어떤 의미인지 알게 하는 것이 목적
